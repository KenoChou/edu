# Edu Mini Program

一个基于微信小程序的教育业务示例项目，覆盖家长端与教师端的核心页面流程，包含登录、课程浏览、订单管理、个人中心等模块。

## 功能概览

- **双角色视角**：支持家长版 / 教师版切换。
- **全局状态管理**：基于轻量 store 统一管理会话、用户资料、订单等状态。
- **登录与鉴权链路**：请求层统一携带 token / role，401 场景统一清理会话并跳转登录。
- **页面导航联动**：首页、我的、订单等页面支持业务入口互相跳转。
- **模块化接口封装**：`services/api.js` + `utils/request.js` 统一请求调用。

## 项目结构

```text
.
├── app.js                     # 小程序入口，初始化全局 store
├── app.json                   # 页面路由与全局配置
├── config.js                  # 环境配置（baseUrl / appId）
├── services/
│   └── api.js                 # 业务 API 封装
├── utils/
│   ├── request.js             # 请求封装与鉴权处理
│   ├── state.js               # 状态常量、默认模型、订单工具方法
│   └── store.js               # 全局可订阅状态管理
└── pages/                     # 页面目录（首页、登录、我的、订单等）
```

## 核心状态模型

全局状态位于 `utils/store.js`，状态定义位于 `utils/state.js`：

- `session`: token、角色、登录状态
- `userProfile`: 用户资料与认证状态
- `orders`: 订单列表

支持：

- `init()`：启动恢复本地缓存
- `setSession()` / `setUserProfile()` / `setOrders()`：按域更新
- `subscribe()`：页面订阅状态变化
- `clearSession()`：退出登录 / 鉴权失效统一清理

## 开发与运行

1. 使用微信开发者工具打开项目根目录。
2. 在 `project.config.json` 中确认项目配置。
3. 根据环境调整 `config.js` 中的 `ENV` 与 `baseUrl`。
4. 编译后可从首页进入业务流程。

> 当前项目中部分登录与业务数据为 mock 演示逻辑，可按后端接口逐步替换。

## 主要业务流

- 登录页：手机号 + 验证码（示例）-> 写入全局会话 -> 跳转首页
- 我的页：根据角色展示不同菜单 -> 点击进入对应功能页
- 订单页：按标签筛选订单 -> 发起支付/查看合同跳转
